# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, email us directly at **security@qivam.com**. Include as much detail as possible:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix if you have one

You will receive an acknowledgement within 48 hours. We will keep you updated as we investigate and address the issue.

## Scope

The following are in scope:

- The Qivam API (authentication, authorisation, data exposure)
- Infrastructure configuration
- Dependency vulnerabilities with a direct exploit path

The following are out of scope:

- Denial of service attacks
- Social engineering
- Issues in third-party services we depend on (report those upstream)

## Disclosure Policy

We follow coordinated disclosure. Once a fix is in place, we will publish a summary of the vulnerability and credit the reporter (unless you prefer to remain anonymous).
